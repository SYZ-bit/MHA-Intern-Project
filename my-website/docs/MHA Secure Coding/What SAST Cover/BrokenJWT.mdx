# Broken JSON Web Token

## Description

Broken JSON Web Token (JWT) attacks are a type of API vulnerability that fall under the broader category of Broken Authentication in the OWASP Top 10 security risks. These attacks occur when JWT-based authentication is improperly implemented, allowing attackers to manipulate tokens and impersonate legitimate users of the web appplication.

JWT is an open internet standard representing claims to be transferred securely, with optional JSON Web Signature (JWS) and JSON Web Encryption (JWE), between parties as a JSON payload. Due to their lightweight structure and digital signature enabling secure verification, JWTs have been widely adopted in commercial applications.

While JWTs have become a popular choice for session and token-based authentication, their effectiveness depends heavily on correct implementation. In many cases, traditional session management may be more suitable. Unfortunately, JWTs are frequently misconfigured or insecurely used, making them an attractive target for attackers seeking to exploit authentication flaws.

## Impact

If an attacker successfully carries out a Broken JWT attack, they can gain the same level of access as the user they are impersonating. Depending on the user’s privileges and how the application is configured, this could allow the attacker to access all data within the application, obtain administrative control, and potentially compromise the application's confidentiality, integrity, and availability.
